# Experiment Ledger

One entry per run, **newest first**. Written and updated by the `experiment`
skill. A run with no verdict is unfinished — fill it in or mark it abandoned.

Copy this template for each new run:

```
## <run-id>  —  YYYY-MM-DD
**Goal:** <one line; link to plan/hypothesis>
**Commit:** <revision; note if working tree was dirty>
**Config:** <key hyperparameters or config path/hash>
**Seed(s):** <...>
**Data:** <dataset / split / version or hash>
**Environment:** <hardware / env hash, if relevant>
**Metrics:** <primary + guardrail metrics, with values>
**Artifacts:** <path to checkpoints / logs / outputs>
**Verdict:** success | fail | inconclusive — <one-line interpretation>
```
